📖 Quick Start Guide

▼

🚀 Launch the System

  1. Start the Evaluation Head: 
    # Start with auto-discovery enabled
sky launch -c eval-head configs/eval_head.yaml \
  --env GAME_SERVER_PREFIX=exp-game \
  --env GAME_SERVER_PORT=8081
  2. Access the Dashboard: 
    # Get the dashboard URL
sky status --endpoint eval-head 8080

# Open in browser: http://<endpoint>/dashboard
  3. Launch Game Servers: 
    # Option A: As regular clusters (no eval head endpoint needed!)
sky launch -c exp-game-1 configs/game_server.yaml
sky launch -c exp-game-2 configs/game_server.yaml
sky launch -c exp-game-3 configs/game_server.yaml

# Option B: As SkyServe service (with autoscaling)
sky serve up -n exp-game-svc configs/game_server.yaml
  4. Monitor Status: 
    sky status         # Check all clusters
sky serve status   # Check services

🔄 Restart & Update

To update code without terminating clusters:

# Restart eval head
sky cancel -ay eval-head
sky exec eval-head configs/eval_head.yaml \
  --env GAME_SERVER_PREFIX=exp-game \
  --env GAME_SERVER_PORT=8081

# Restart game servers
sky cancel -ay exp-game-1
sky exec exp-game-1 configs/game_server.yaml

# For services
sky serve update exp-game-svc configs/game_server.yaml
💡 Tips:
  • Game servers are auto-discovered - no need to pass eval head endpoint!
  • Both clusters and SkyServe services with prefix exp-game-* are discovered
  • Episodes have randomized lengths (±30% variation + 1% early termination)
  • The eval head actively controls game servers via HTTP endpoints
  • SkyServe provides automatic failover and autoscaling
  • The dashboard auto-refreshes with real-time metrics
  • Discovery checks for new clusters/services every 30 seconds
  • Game servers can run on spot instances for 70-90% cost savings
